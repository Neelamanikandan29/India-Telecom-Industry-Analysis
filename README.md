# 📊 India Telecom Performance & Growth Analytics Dashboard

## 📌 Project Overview

This project focuses on analyzing the performance and growth trends of the Indian telecom sector using a structured dataset. The dashboard provides insights into revenue trends, subscriber growth, network performance, and regional distribution, helping stakeholders make data-driven decisions.

The dataset used in this project is a **calibrated synthetic dataset (6000 rows)** inspired by industry sources such as **TRAI, IMARC, and Mordor Intelligence**.

---

## 🎯 Problem Statement

The Indian telecom industry is highly competitive and rapidly evolving. Organizations need a centralized analytical solution to:

- Monitor revenue growth across telecom operators  
- Analyze subscriber trends and churn patterns  
- Evaluate network performance (signal strength, coverage)  
- Identify regional growth opportunities  
- Improve decision-making through data visualization  

---

## 💡 Project Objective

- Clean and preprocess telecom data efficiently  
- Transform raw data into a structured analytical model  
- Build an interactive dashboard for performance tracking  
- Derive actionable insights from telecom metrics  

---

## 🗂 Dataset Information

- **Source**: TRAI / IMARC / Mordor Intelligence (Calibrated Synthetic Data)  
- **Total Rows**: 6000  

### Key Fields:

- Telecom Operator  
- Revenue  
- Subscriber Count  
- Region / State  
- Network Type (4G/5G)  
- Signal Strength (dBm)  
- Customer Satisfaction Index  

---

## 🧹 Data Cleaning (Advanced Excel)

Data preprocessing was performed using **Microsoft Excel**:

- Removed duplicate records  
- Handled missing values  
- Standardized column formats  
- Cleaned inconsistent text entries  
- Created calculated columns (e.g., signal strength categories)  

---

## 🔄 Data Transformation (Power BI - Power Query Editor)

The dataset was further processed using **Power BI ETL (PQE)**:

- Data type transformations  
- Column splitting and merging  
- Filtering irrelevant records  
- Creating derived columns  
- Building relationships between tables  
- Optimizing data model for performance
